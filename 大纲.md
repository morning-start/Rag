## 第一阶段：认知篇 — RAG 是什么，为什么需要它

### 第1章：RAG 概述与核心价值

#### 1.1 大模型的局限性
- 知识截止（Knowledge Cutoff）问题
- 幻觉（Hallucination）的三种类型
  - 输入冲突幻觉：回答与用户输入矛盾
  - 上下文冲突幻觉：回答与给定上下文矛盾
  - 事实冲突幻觉：回答与客观事实矛盾
- 领域知识缺失与数据隐私困境

#### 1.2 RAG 的核心思想
- 开卷考试比喻：检索（Retrieval）→ 增强（Augmentation）→ 生成（Generation）
- 从 Lewis et al. (NeurIPS 2020) 论文到工业标配的演进历程
- RAG 解决的核心痛点：幻觉降 70%-90%、知识实时更新、私有数据安全

#### 1.3 RAG vs 提示工程 vs 微调
- 三者对比表（成本、效果、适用场景）
- 什么时候选 RAG，什么时候选微调
- 最佳实践：RAG + 微调结合使用

### 第2章：RAG 技术架构演进（五代范式）

#### 2.1 Naive RAG（2023 年前，朴素 RAG）
- 简单流水线：用户提问 → 向量检索 Top-K → 拼接上下文 → LLM 生成
- 核心问题：检索精度低、缺乏上下文过滤、无自我校正

#### 2.2 Advanced RAG（2023-2024，高级 RAG）
- 预检索优化：查询重写、多查询扩展
- 后检索优化：重排序、上下文压缩
- 混合检索（Hybrid Search）：BM25 + 向量检索 + RRF 融合

#### 2.3 Modular RAG（2024，模块化 RAG）
- 可插拔、可组合的流水线架构
- 检索器池（Retriever Pool）：多路并行检索
- 输入/输出验证模块

#### 2.4 Graph RAG（2024-2025，图谱 RAG）
- Microsoft Research 提出，知识图谱替代纯向量检索
- 全局查询：利用图谱社区结构做抽象摘要
- 局部查询：从实体出发做多跳推理
- 解决传统 RAG 在"全局性问题"上的根本缺陷

#### 2.5 Agentic RAG（2025-2026，智能体 RAG）
- 自适应检索：判断是否需要检索、检索几轮
- 工具调用：搜索引擎、数据库、API
- 自我反思（Self-Reflection）：验证检索结果质量，必要时重新检索
- 多步推理：复杂问题拆解为多个子查询

## 第二阶段：基础篇 — 从零搭建你的第一个 RAG 系统

### 第3章：环境准备与工具链

#### 3.1 技术选型概览
- 技术选型：混合方案（底层组件原生 SDK + 编排层 LangGraph）
- 模型选择：OpenAI / Ollama 本地模型 / HuggingFace 开源模型
- 向量数据库对比：Chroma / FAISS / Pinecone / Milvus / Qdrant

#### 3.2 开发环境搭建
- Python 3.10+ 环境配置（Conda / venv）
- 核心依赖安装：httpx、Chroma、FAISS、LangGraph、Ollama
- Ollama 本地部署与模型拉取（llama3、qwen、gemma 等）
- 项目目录结构规范

### 第4章：数据处理与索引管线（Indexing Pipeline）

#### 4.1 文档加载（Document Loading）
- 多格式解析：PDF（PyMuPDF/Unstructured）、Word（python-docx）、CSV/Excel（openpyxl）、JSON、HTML（BeautifulSoup）
- HTML 加载器：网页内容提取与清洗
- Excel 加载器：多 Sheet 表格解析与结构化处理
- 目录批量加载与自定义加载器

#### 4.2 文档分块（Chunking）策略
- 固定大小分块：按字符数/Token 数滑动窗口切分
- 递归分块：按分隔符优先级递归切分（自定义实现）
- 语义分块：Embedding 相似度识别语义边界
- 父子分块（Small-to-Big）：小块检索 + 大块上下文
- 滑动窗口（Overlap）：保留跨块连续性
- 策略对比与选型建议表

#### 4.3 向量嵌入（Embedding）
- 什么是 Embedding：从文本到向量的映射
- 开源 vs 商用 Embedding 模型选择（OpenAI text-embedding-ada-002 / BGE / e5 / m3e）
- 批量嵌入与缓存优化（避免重复嵌入已处理文档）

#### 4.4 向量存储与索引
- 向量数据库核心概念：HNSW、IVF、余弦距离
- Chroma 本地持久化存储
- FAISS 内存索引
- 使用 Pinecone / Milvus 作为生产级存储

### 第5章：检索与生成管线（Retrieval & Generation Pipeline）

#### 5.1 检索模块
- 语义检索（Dense Retrieval）：向量相似度搜索
- 关键词检索（Sparse Retrieval / BM25）：精确匹配
- 混合检索（Hybrid Search）：BM25 + 向量 + RRF 融合排序
- MMR（最大边际相关性）去重

#### 5.2 查询增强（Query Enhancement）
- 查询重写（Rewrite）：口语→正式表达
- 多查询扩展（Multi-Query）：多角度并行检索
- HyDE（假设文档嵌入）：生成伪文档后进行检索
- 子问题分解：复杂问题拆解为多个可独立检索的子问题

#### 5.3 重排序（Reranking）
- RRF（倒数排名融合）：轻量多路结果融合
- Cross-Encoder 深度精排（BGE-Reranker 等）
- 场景选择：高精准场景用 Cross-Encoder，高并发场景用 RRF

#### 5.4 生成模块
- 上下文组装：将检索结果注入 Prompt
- 提示词工程：引用溯源、指令约束、结构化输出
- 生成策略：温度控制（Temperature）、Top-P 采样
- Self-RAG：自反思机制，降低幻觉
- 领域微调：使 LLM 学会"如何使用材料"

#### 5.5 完整 RAG 代码实战（→ 第5A章）
- 从加载文档 → 分块 → 嵌入 → 存储 → 检索 → 生成 全流程
- LangGraph StateGraph 实现完整 RAG 工作流
- 返回检索来源（Source Documents）与引用
- **详细代码实现见 [第5A章 LangGraph 完整 RAG 实战](book/ch05a-LangGraph完整RAG实战.md)**

## 第三阶段：进阶篇 — 提升 RAG 系统质量

### 第6章：检索质量优化

#### 6.1 分块参数调优
- Chunk Size & Overlap 的实验设计方法
- 不同文档类型的最佳参数推荐

#### 6.2 元数据过滤
- 按时间、来源、作者、类型等字段精确过滤
- 结构化检索与混合过滤结合

#### 6.3 上下文压缩
- 冗余过滤：MMR 保持多样性
- 摘要压缩：长文档先摘要再检索
- LLM as Judge：智能筛选相关片段
- Lost-in-the-Middle 排列优化：关键信息置于上下文首尾
- 检索精度提升（混合检索 RRF + Cross-Encoder 重排序）*→ 第5章深化延伸*

### 第7章：生成质量优化

#### 7.1 Prompt 工程深度实践
- System Prompt 设计模板
- Few-shot 示例增强
- 强制引用格式与结构化输出

#### 7.2 幻觉抑制策略
- 引用溯源（Citation）：回答必须标注来源片段
- 轻量级置信度评估与独立多轮验证：让模型评估自身回答的可信度，独立多轮校验是否基于上下文

#### 7.3 多轮对话增强
- 对话历史记忆（Conversation Buffer Memory）
- 上下文指代消解（指代"它""这个"等代词）

#### 7.4 训练层方案（进阶选读）
- SFT（监督微调）让模型学会"只基于上下文回答"
- DPO / RLHF 强化忠实度偏好
- 训练数据构建：基于 RAG 输出的高质量问答对

### 第8章：评估体系（Evaluation）

#### 8.1 检索评估指标
- Recall@K、Precision@K
- MRR（Mean Reciprocal Rank）、NDCG

#### 8.2 生成评估指标
- Faithfulness（忠实度）：回答是否基于检索内容
- Answer Relevance（回答相关性）
- Context Precision / Context Recall

#### 8.3 评估工具与框架
- RAGAS（RAG Assessment）：自动化评估框架
- TruLens 评估
- 人工评估集构建方法
