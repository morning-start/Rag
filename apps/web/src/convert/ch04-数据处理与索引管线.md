# 第4章 数据处理与索引管线

上一章我们完成了环境搭建和技术选型。本章将深入 RAG 系统的第一条流水线——索引管线（Indexing Pipeline），包含四个核心环节：文档加载、文档分块、向量嵌入、向量存储。每个环节的质量直接决定了后续检索的上限。所有代码均使用**原生 Python 库**实现，不依赖 LangChain 或任何封装框架。

---

## 4.1 文档加载

文档加载是索引管线的起点。真实场景中的数据来源多样——PDF 合同、Word 文档、CSV 表格、JSON 日志、HTML 网页、Excel 报表。我们将为每种格式选择最合适的原生 Python 库，统一输出结构：`{"text": str, "metadata": dict}`。

### 4.1.1 统一加载模式

所有文档加载器遵循同一接口模式：

```python
def load_<format>(source: str, **kwargs) -> List[Dict]:
    """加载指定格式文档，返回 [{"text": str, "metadata": dict}, ...]"""
```

各格式的推荐库与关键参数：

| 格式 | 推荐库 | 关键参数 | 说明 |
|------|--------|---------|------|
| PDF | PyMuPDF (fitz) | — | 按页提取文字，扫描版需 OCR（第6章）|
| Word (.docx) | python-docx | — | 按段落返回，含 style 元数据 |
| CSV | 内置 csv | content_column | 每行一个文档，其他列作为元数据 |
| JSON | 内置 json | content_field, is_list | 支持顶层列表或指定字段 |
| HTML | httpx + BeautifulSoup | content_tag | 自动移除导航/脚本等噪音标签 |
| Excel (.xlsx) | openpyxl | sheet_name, content_columns | 支持多工作表，读取公式缓存值 |

PDF 是企业知识库最常见的格式，我们选用 **PyMuPDF（fitz）** 作为解析引擎——它在纯文字 PDF 上准确率达 82%，处理速度超过 50 页/秒，且无需额外依赖。（来源: [01-PDF解析工具横评与选型.md](reference/05-数据处理与索引/01-PDF解析工具横评与选型/)）

```python
import fitz
from pathlib import Path
from typing import List, Dict

def load_pdf(file_path: str) -> List[Dict]:
    if not Path(file_path).exists():
        raise FileNotFoundError(f"文件不存在: {file_path}")
    docs = []
    doc = fitz.open(file_path)
    for page_num, page in enumerate(doc):
        text = page.get_text()
        if text.strip():
            docs.append({
                "text": text.strip(),
                "metadata": {"source": file_path, "page": page_num + 1, "file_type": "pdf"}
            })
    doc.close()
    return docs
```

对于扫描版 PDF（图片型），PyMuPDF 无法提取文字，需要集成 OCR 方案（如 PaddleOCR 或 Tesseract），这属于进阶话题，将在第6章展开。

### 4.1.2 其他格式快速调用

Word、CSV、JSON 等其他格式遵循同一接口模式。每个格式的参数已在 4.1.1 的表格中给出。以 JSON 为例（需处理嵌套字段和列表结构，最具代表性）：

```python
import json

def load_json(file_path: str, content_field: str = "text", is_list: bool = True) -> List[Dict]:
    if not Path(file_path).exists():
        raise FileNotFoundError(f"文件不存在: {file_path}")
    with open(file_path, "r", encoding="utf-8") as f:
        data = json.load(f)
    docs = []
    items = data if is_list else data.get("items", [])
    for idx, item in enumerate(items):
        if isinstance(item, dict) and content_field in item:
            text = item[content_field]
            if isinstance(text, str) and text.strip():
                metadata = {"source": file_path, "file_type": "json", "index": idx}
                for k, v in item.items():
                    if k != content_field:
                        metadata[k] = str(v)
                docs.append({"text": text.strip(), "metadata": metadata})
    return docs
```

Word（python-docx）按段落遍历并记录 style 元数据，逻辑与 PDF 类似，不再赘述。CSV 使用内置 `csv.DictReader` 按行读取并将其他列自动归档到元数据。

HTML 解析的核心挑战在于去除导航栏、广告、侧边栏等噪音内容。我们选用 **httpx**（支持 HTTP/2）+ **BeautifulSoup**（lxml 解析器）的组合。

```python
import httpx
from bs4 import BeautifulSoup

def load_html(url: str, content_tag: str = "article") -> List[Dict]:
    response = httpx.get(url, follow_redirects=True, timeout=30)
    response.raise_for_status()
    soup = BeautifulSoup(response.text, "lxml")
    for tag in soup(["script", "style", "nav", "footer", "header", "aside"]):
        tag.decompose()
    content = soup.find(content_tag) or soup.find("body") or soup
    text = content.get_text(separator="\n", strip=True)
    if not text:
        return []
    title = soup.find("title")
    return [{"text": text, "metadata": {
        "source": url, "file_type": "html",
        "title": title.get_text(strip=True) if title else "",
        "load_date": __import__("datetime").datetime.now().isoformat()
    }}]
```

注意事项：JavaScript 渲染页面需无头浏览器；生产环境应设置 User-Agent 和请求间隔；通用提取可考虑 `trafilatura`。

Excel（.xlsx）可包含多工作表、合并单元格和公式。选用 **openpyxl** 解析：

```python
from openpyxl import load_workbook

def load_excel(file_path: str, sheet_name: str = None,
               content_columns: List[str] = None) -> List[Dict]:
    if not Path(file_path).exists():
        raise FileNotFoundError(f"文件不存在: {file_path}")
    wb = load_workbook(file_path, read_only=True, data_only=True)
    ws = wb[sheet_name] if sheet_name and sheet_name in wb.sheetnames else wb.active
    docs, rows = [], list(ws.iter_rows(values_only=True))
    if not rows:
        wb.close(); return []
    headers = [str(h) if h else f"col_{i}" for i, h in enumerate(rows[0])]
    for row_idx, row in enumerate(rows[1:], start=2):
        row_data = dict(zip(headers, row))
        if not any(v is not None and str(v).strip() for v in row):
            continue
        if content_columns:
            parts = [f"{col}: {row_data.get(col, '')}" for col in content_columns if col in row_data]
        else:
            parts = [f"{k}: {v}" for k, v in row_data.items() if v is not None]
        docs.append({"text": "\n".join(parts).strip(), "metadata": {
            "source": file_path, "file_type": "xlsx", "sheet": ws.title, "row": row_idx
        }})
    wb.close()
    return docs
```

注意事项：`data_only=True` 读取公式缓存值（可能为 None）；`read_only=True` 流式模式适合大文件；合并单元格仅返回左上角值。

### 4.1.3 目录批量加载

```python
from glob import glob

def load_directory(dir_path: str, pattern: str = "**/*.pdf") -> List[Dict]:
    if not Path(dir_path).is_dir():
        raise NotADirectoryError(f"目录不存在: {dir_path}")
    all_docs = []
    for fp in glob(os.path.join(dir_path, pattern), recursive=True):
        ext = Path(fp).suffix.lower()
        try:
            if ext == ".pdf":    all_docs.extend(load_pdf(fp))
            elif ext == ".docx": all_docs.extend(load_docx(fp))
            elif ext == ".csv":  all_docs.extend(load_csv(fp))
            elif ext == ".json": all_docs.extend(load_json(fp))
            elif ext == ".xlsx": all_docs.extend(load_excel(fp))
            else: print(f"  [跳过] 不支持的格式: {fp}")
        except Exception as e:
            print(f"  [错误] {fp}: {e}")
    return all_docs
```

## 4.2 文档分块

分块是索引管线中**最被低估但影响最大**的环节。研究表明，分块策略解释了约 80% 的检索质量差异。（来源: [04-生产级RAG落地12组件.md](reference/02-RAG核心原理/04-生产级RAG落地12组件/)）

分块的本质是将长文档切成语义完整的小片段，既能被 Embedding 模型有效编码，又能在检索时精准匹配用户查询。

### 4.2.1 固定大小分块

```python
def fixed_size_split(text: str, chunk_size: int = 500, overlap: int = 50) -> List[str]:
    if len(text) <= chunk_size:
        return [text]
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks
```

优点：实现简单，处理速度快。缺点：极易在词中间截断，对中文尤其不友好。

### 4.2.2 递归字符分块（推荐默认方案）

递归分块是**生产环境中最推荐的默认策略**。它按分隔符优先级递归尝试，优先在语义边界处切分。分隔符优先级：`"\n\n"`（段落）→ `"\n"`（行）→ `"。"`（句号）→ `"！"` → `"？"` → `"；"` → `"，"` → `" "`（空格）。

```python
def recursive_split(
    text: str,
    chunk_size: int = 500,
    overlap: int = 50,
    separators: List[str] = None
) -> List[str]:
    if separators is None:
        separators = ["\n\n", "\n", "。", "！", "？", "；", "，", " ", ""]
    if separators[-1] == "":
        return fixed_size_split(text, chunk_size, overlap)
    separator = separators[0]
    remaining_separators = separators[1:]
    if separator not in text:
        return recursive_split(text, chunk_size, overlap, remaining_separators)
    parts = text.split(separator)
    chunks = []
    current_chunk = ""
    for part in parts:
        new_chunk = current_chunk + (separator if current_chunk else "") + part
        if len(new_chunk) <= chunk_size:
            current_chunk = new_chunk
        else:
            if current_chunk:
                chunks.append(current_chunk)
            sub_chunks = recursive_split(part, chunk_size, overlap, remaining_separators)
            if sub_chunks:
                chunks.extend(sub_chunks[:-1])
                current_chunk = sub_chunks[-1] if sub_chunks else ""
    if current_chunk:
        chunks.append(current_chunk)
    return chunks

text = "这是第一段。\n\n这是第二段，内容比较长需要详细说明一些技术细节。"
chunks = recursive_split(text, chunk_size=30, overlap=5)
for i, chunk in enumerate(chunks):
    print(f"Chunk {i} ({len(chunk)}字): {chunk}")
```

> **分隔符必须拆分为独立元素**：`"。"` 和 `"！"` 不能合并为 `"。！"`，因为后者仅匹配连续出现的情况，而实际文本中它们是单独出现的，合并会导致实际上跳过了句末级别的切分。

### 4.2.3 对文档列表进行分块

```python
def split_documents(documents: List[Dict], chunk_size: int = 500, overlap: int = 50) -> List[Dict]:
    chunks = []
    for doc in documents:
        sub_chunks = recursive_split(doc["text"], chunk_size=chunk_size, overlap=overlap)
        for idx, text in enumerate(sub_chunks):
            chunks.append({
                "text": text,
                "metadata": {**doc["metadata"], "chunk_index": idx, "total_chunks": len(sub_chunks)}
            })
    return chunks

documents = load_directory("data/raw/", pattern="**/*.pdf")
chunks = split_documents(documents, chunk_size=500, overlap=50)
```

### 4.2.4 语义分块

语义分块通过 Embedding 计算相邻句子的相似度，在主题转变处插入边界。质量最高但成本也更高。

```python
import numpy as np
from sentence_transformers import SentenceTransformer

def semantic_split(
    text: str,
    model: SentenceTransformer = None,
    threshold: float = 0.3,
    min_chunk_size: int = 100
) -> List[str]:
    if model is None:
        model = SentenceTransformer("all-MiniLM-L6-v2")
    sentences = [s.strip() for s in text.replace("?", "?\n").replace("!", "!\n").split("。") if s.strip()]
    if len(sentences) <= 1:
        return [text] if text.strip() else []
    embeddings = model.encode(sentences)
    chunks = []
    current_chunk = [sentences[0]]
    for i in range(1, len(sentences)):
        sim = np.dot(embeddings[i], embeddings[i-1]) / (
            np.linalg.norm(embeddings[i]) * np.linalg.norm(embeddings[i-1]) + 1e-8
        )
        if sim < threshold and len("".join(current_chunk)) >= min_chunk_size:
            chunks.append("".join(current_chunk))
            current_chunk = [sentences[i]]
        else:
            current_chunk.append(sentences[i])
    if current_chunk:
        chunks.append("".join(current_chunk))
    return chunks
```

### 4.2.5 父子分块

父子分块的核心思想是"小块检索 + 大块生成"：子块（~200 tokens）用于精准向量匹配，父块（~1000 tokens）提供完整上下文给 LLM。通过**字符偏移量**建立子块到父块的可信映射。

```python
def parent_child_split(
    documents: List[Dict],
    child_size: int = 200,
    parent_size: int = 1000,
    child_overlap: int = 20,
    parent_overlap: int = 100
) -> tuple:
    all_parent_chunks = []
    all_child_chunks = []

    for doc in documents:
        text = doc["text"]

        parent_offsets = []
        start = 0
        while start < len(text):
            end = min(start + parent_size, len(text))
            parent_offsets.append((start, end))
            start += parent_size - parent_overlap
        for idx, (p_start, p_end) in enumerate(parent_offsets):
            all_parent_chunks.append({
                "text": text[p_start:p_end],
                "metadata": {**doc["metadata"], "chunk_type": "parent",
                             "char_start": p_start, "char_end": p_end, "chunk_index": idx}
            })
        child_offsets = []
        start = 0
        while start < len(text):
            end = min(start + child_size, len(text))
            child_offsets.append((start, end))
            start += child_size - child_overlap
        for idx, (c_start, c_end) in enumerate(child_offsets):
            all_child_chunks.append({
                "text": text[c_start:c_end],
                "metadata": {**doc["metadata"], "chunk_type": "child",
                             "char_start": c_start, "char_end": c_end, "chunk_index": idx}
            })
    mapping = {}
    for c_idx, child in enumerate(all_child_chunks):
        c_source = child["metadata"].get("source", "")
        c_start, c_end = child["metadata"]["char_start"], child["metadata"]["char_end"]
        best_p_idx = None
        best_overlap = -1
        for p_idx, parent in enumerate(all_parent_chunks):
            if parent["metadata"].get("source", "") != c_source:
                continue
            p_start, p_end = parent["metadata"]["char_start"], parent["metadata"]["char_end"]
            overlap_len = max(0, min(c_end, p_end) - max(c_start, p_start))
            if overlap_len > best_overlap:
                best_overlap = overlap_len
                best_p_idx = p_idx
        if best_p_idx is not None:
            mapping[c_idx] = best_p_idx

    return all_parent_chunks, all_child_chunks, mapping
```

**为什么字符偏移量比 source + page 更可靠？** 同一页可能被切成多个父块和子块，仅靠页码无法确定对应关系；CSV、JSON 等格式没有"页"概念。字符偏移量利用区间包含关系确定映射，不受文档类型限制。

## 4.3 向量嵌入（Embedding）

分块完成后，下一步是将文本片段转化为向量表示——即 Embedding。Embedding 模型将一段文本映射为一个高维数值向量，语义相近的文本在向量空间中距离更近。

### 4.3.1 什么是 Embedding

Embedding 的本质是"把文本的含义压缩成一组数字"：

```python
import numpy as np
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")
vec1, vec2, vec3 = (
    model.encode("机器学习是 AI 的子领域"),
    model.encode("深度学习是人工智能的一种技术"),
    model.encode("今天天气很好"),
)

def cos_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b) + 1e-8)

print(f"维度: {len(vec1)}")
print(f"AI 句子相似度: {cos_sim(vec1, vec2):.3f}")   # ~0.85
print(f"AI vs 天气相似度: {cos_sim(vec1, vec3):.3f}")  # ~0.30
```

这就是向量检索能"理解"语义的基础——语义相近的段落向量距离更近，无关段落距离更远。

### 4.3.4 Embedding 缓存优化

Embedding 计算是索引管线中最耗时的环节。以文本哈希值为键缓存已计算的向量，下次遇到相同文本时直接从缓存读取，跳过模型推理。由于 Embedding 是确定性的（相同输入永远产生相同输出），缓存不会引入精度损失。

**基于 SQLite 的持久化缓存（推荐生产使用）**：

```python
import hashlib, sqlite3, json
from typing import Optional

class EmbeddingCache:
    def __init__(self, db_path: str = "./cache/embedding_cache.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.execute(
            "CREATE TABLE IF NOT EXISTS embedding_cache "
            "(text_hash TEXT PRIMARY KEY, model_name TEXT, dimension INT, vector TEXT)")

    def _hash_text(self, text: str) -> str:
        return hashlib.sha256(text.encode("utf-8")).hexdigest()

    def get(self, text: str, model_name: str) -> Optional[list]:
        text_hash = self._hash_text(text)
        row = self.conn.execute(
            "SELECT vector FROM embedding_cache WHERE text_hash=? AND model_name=?",
            (text_hash, model_name)
        ).fetchone()
        return json.loads(row[0]) if row else None

    def put(self, text: str, model_name: str, vector: list):
        text_hash = self._hash_text(text)
        self.conn.execute("INSERT OR REPLACE INTO embedding_cache VALUES (?,?,?,?)",
                          (text_hash, model_name, len(vector), json.dumps(vector)))
        self.conn.commit()

    def close(self):
        self.conn.close()

def cached_batch_encode(texts: List[str], model_name: str = "all-MiniLM-L6-v2"):
    cache = EmbeddingCache()
    model = SentenceTransformer(model_name)
    results, to_encode = [], []
    for i, text in enumerate(texts):
        cached = cache.get(text, model_name)
        results.append(cached)
        if cached is None:
            to_encode.append(i)
    if to_encode:
        new_emb = model.encode([texts[i] for i in to_encode], normalize_embeddings=True)
        for idx, i in enumerate(to_encode):
            vec = new_emb[idx].tolist()
            results[i] = vec
            cache.put(texts[i], model_name, vec)
    cache.close()
    return np.array(results)
```

**注意事项**：更换模型后必须清空缓存（通过 `model_name` 字段隔离不同模型，建议包含版本哈希）；SHA-256 碰撞概率可忽略；建议定期清理膨胀的缓存文件；计算哈希前对文本规范化（如 `strip()`）避免因空白差异导致缓存未命中。

### 4.4.1 距离度量

| 度量 | 特点 | 适用场景 |
|------|------|---------|
| 余弦相似度（Cosine） | 衡量方向相似性，忽略长度 | **最常用，文本检索首选** |
| 欧氏距离（L2） | 衡量绝对距离 | 特征值范围一致时 |
| 点积（Dot Product） | 衡量方向+长度 | 向量归一化后等价于余弦 |

import chromadb
from chromadb.utils import embedding_functions

def build_index(data_dir: str, persist_dir: str = "./vectorstore/chroma_db"):
    documents = load_directory(data_dir, pattern="**/*.pdf")
