## 核心内容

### 为什么 RAG 还没被长上下文取代？

| 维度 | 长上下文直接塞 | RAG 检索 |
|------|--------------|---------|
| 响应延迟 | 高 | 低 |
| 推理成本 | 极高 | 可控 |
| 知识更新 | 需重新推理 | 只更新知识库 |
| 精确度 | 随文档数量下降 | 高 |
| 适用规模 | <100万字 | 亿级文档 |

### RAG 技术演进的五个阶段

#### 阶段一：Naive RAG（2023 年主流）
最简单的实现：
1. 文档切片 → 向量化 → 存入向量数据库
2. 用户问题 → 向量检索 → Top-K 片段
3. 片段 + 问题 → LLM → 回答
问题：切片粒度难以兼顾，相关性检索不精准，没有多轮对话记忆

#### 阶段二：Advanced RAG（2024）
引入检索前/后优化：
- **Pre-retrieval**: 查询重写、HyDE 假设文档生成
- **Retrieval**: 混合检索（BM25+向量）、语义分块
- **Post-retrieval**: 重排序、上下文压缩
召回率提升 30-50%

#### 阶段三：Modular RAG（2024-2025）
模块化设计，每个步骤可插拔替换：
`[查询理解] → [检索策略路由] → [多源检索] → [结果聚合] → [生成]`

#### 阶段四：自适应 RAG（2025）
模型自主判断是否需要检索：
- 简单问题 → 直接回答不检索
- 实时数据问题 → 触发检索
- 多步推理问题 → 迭代检索

#### 阶段五：Agentic RAG（2026 当前主流）
RAG 成为 Agent 的记忆模块：
- **主动检索**: Agent 自主决定何时、检索什么
- **迭代精炼**: 检索→评估→再检索
- **多源异构**: 内部文档+实时搜索+结构化数据库
- **图检索增强**: 知识图谱多跳推理

### 2026 年 RAG 五大新范式

1. **Self-RAG**: 反思 Token（[Retrieve]/[IsRel]/[IsSup]），幻觉率降低 40%
2. **GraphRAG**: 知识图谱支持多跳推理
3. **混合检索 2.0**: 结构化数据+时序+跨模态
4. **RAG-as-Memory**: Agent 长期记忆模块
5. **RAG + MCP**: 通过标准协议被 Agent 调用
