# 生产级 RAG 落地全指南：12 个核心组件

**来源**: CSDN - hubu2k
**链接**: https://blog.csdn.net/huhu2k/article/details/159087522
**类型**: 中文系统教程（完整架构拆解）
**适用章节**: Ch4-Ch11

## 内容总览

文章从文档摄入到效果评估，拆解了生产级 RAG 的 12 个核心组件：

### 1. Document Ingestion（文档摄入）
- 多格式解析（PDF、Word、扫描件 OCR）
- 增量同步机制
- 元数据留存
- **最佳实践**：Unstructured + LangChain Loaders + PyMuPDF

### 2. Data Preprocessing（数据预处理）
- 噪声过滤（乱码、符号、广告导航）
- 文本标准化（大小写、日期格式、术语统一）
- 分词与停用词移除
- **最佳实践**：按业务场景定制预处理规则

### 3. Chunking（分块）⭐ 最被低估的核心环节
分块策略解释了约 **80% 的检索质量差异**

| 策略 | 忠实度 |
|------|--------|
| 固定大小分块 | 0.47-0.51 |
| 递归分块 | 0.58-0.64 |
| 语义分块 | 0.79-0.82 |
| 父子分块（Small-to-Big） | 0.83-0.89 |

**最佳实践**：
- 放弃固定大小，采用语义分块
- 父子块架构：子块检索 + 父块生成
- 文档适配：FAQ 128-256 tokens / 技术文档 512-1024 tokens
- 10%-20% 块重叠

### 4. Embedding Generation（嵌入生成）
- 场景与模型匹配
- 嵌入一致性（生成、检索、重排序用同一模型）
- 向量归一化
- 缓存与增量生成

### 5. Vector Database（向量数据库）
- 选型：Chroma/FAISS（轻量）vs Pinecone/Milvus（生产）
- 索引配置：HNSW 平衡速度与精度
- 元数据过滤
- 分片、副本、备份

### 6. Retriever（检索器）
- **混合检索**（Hybrid Search）：向量 + BM25
- **多查询检索**（Multi-Query）：多角度拆分
- 动态 Top-K
- 父文档检索

### 7. Ranking（重排序）⭐ 用极小成本实现效果翻倍
- Cross-Encoder（BGE-Reranker / ColBERT）
- Top20-30 → 重排序 → Top5-10
- 自定义加权规则

### 8. Context Assembly（上下文组装）
- 结构化、去冗余、控长度、标来源
- 按相关性排序
- 预留生成空间

### 9. Prompt Engineering（提示词工程）
6 大核心要素：
1. 角色设定
2. 核心规则（必须基于上下文）
3. 格式约束
4. 引用标注要求
5. 异常处理（无相关内容时）
6. 安全护栏

### 10. Answer Generation（回答生成）
- 温度控制
- Self-RAG 自反思
- 引用溯源

### 11. Guardrails & Security（安全护栏）
- 内容安全校验
- Prompt 注入防护
- 输出敏感信息检测

### 12. Evaluation & Iteration（评估与迭代）
- Context Precision / Recall
- Faithfulness
- Answer Relevance
- 数据飞轮
